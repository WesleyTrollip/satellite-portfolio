# OpenAPI Outline: Portfolio Tracker MVP

This is a lightweight outline. The source of truth will be the Swagger/OpenAPI generated by the ASP.NET Core API, but this file pins down conventions early.

## General conventions

- **REST + JSON**
- **ISO-8601 timestamps** (UTC recommended)
- **Money**: decimals in EUR for MVP; do not use floating point.
- **Pagination**: optional for lists; include `limit`/`cursor` later if needed.
- **Errors**: consistent problem details responses (`application/problem+json`).

## Correction semantics (auditable edits)

Avoid silent overwrites for ledger events.

- Trades and cash entries are immutable economic events.
- “Editing” creates an auditable correction:
  - reversal + replacement (preferred), or
  - versioned records

API responses should make corrections visible:
- include fields like `correctionGroupId`, `isCorrectionReversal`, `correctedById`, and/or a `status`.

## Key response shapes (conceptual)

- `PortfolioOverviewDto`
  - `asOf`
  - `cashBalance`
  - `totalValue`
  - `realizedPnlTotal`
  - `unrealizedPnlTotal`
  - `alerts[]`
  - `topHoldings[]`

- `HoldingDto`
  - `instrumentId`, `symbol`, `sector`
  - `quantity`
  - `avgCost` (if average cost method)
  - `lastPrice` + `lastPriceDate` (optional)
  - `marketValue` (optional)
  - `unrealizedPnl` (optional)
  - `allocationPct` (optional)
  - `missingPrice` (bool) + `missingPriceReason` (string)

- `TradeDto`, `CashEntryDto` include correction metadata.
